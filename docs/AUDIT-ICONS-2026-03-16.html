Stage Stock — Audit Rendu d'Icônes React

Audit structuré — 16 mars 2026

1. Résumé Exécutif

Le projet Stage Stock utilise des icônes provenant de la librairie Lucide React. Lors de la migration vers des références de composants React (au lieu de chaînes emoji), certains fichiers rendaient ces composants comme du texte, provoquant le crash React Error #31. Cet audit a identifié et corrigé toutes les instances.

Résultat : 0 crash runtime restant, 2 corrections préventives appliquées.

3. Fichiers Corrigés

3.1 AccessManager.jsx — Crash React #31

  • Ligne 191 : {roleConf.icon} → createElement(roleConf.icon, { size: 18, color: roleConf.color })
  • Ligne 329 : Même pattern corrigé dans le modal d'édition membre
  • Ligne 355 : {rc?.icon || ''} retiré des balises <option> (impossible de rendre des composants dans des options HTML)

3.2 ProfilePage.jsx — Préventif

  • Ligne 27 : Champ icon retiré de SKILL_OPTIONS — stockait une référence composant Lucide inutilisée (dead weight)

3.3 Checklists.jsx — Préventif

  • Ligne 310 : {conf.icon} → {typeof conf.icon === 'string' ? conf.icon : ''} — protection contre une future migration vers des icônes composant

5. Recommandations

  1. Convention à adopter : Toujours utiliser createElement(icon, props) ou <Icon {...props} /> pour rendre des icônes Lucide. Ne jamais insérer {icon} directement dans le JSX.
  2. Balises <option> : Les balises HTML <option> ne supportent que du texte. Toujours vérifier typeof icon === 'string' avant insertion.
  3. Tests futurs : Ajouter un test de non-régression qui vérifie que toute config exportée avec un champ icon de type Function est consommée via createElement.
